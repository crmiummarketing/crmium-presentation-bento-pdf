# CRMiUM Presentation — Bento Style → PDF (no QR)

Скіл генерує фірмові презентації CRMiUM у Bento-дизайні **без QR-кодів** і віддає їх як HTML + готовий **PDF**. Універсальний — будь-який спікер, тема, мова (UA/EN/PL/RU). Підтримує **збереження кількох спікерів**.

Це варіант сусіднього скіла `crmium-presentation-bento`. Відмінності рівно дві:
1. **Жодних QR-кодів** — слайд Q&A лишається як просте «Питання?», слайд Resources стає текстовим (без Telegram-QR).
2. **PDF як повноцінний результат** — після підтвердження спікера скіл сам рендерить PDF через headless Chrome, без ручного Ctrl+P.

## 🛑 АБСОЛЮТНІ ЗАБОРОНИ — ПРОЧИТАЙ ПЕРЕД БУДЬ-ЯКОЮ ДІЄЮ

### ❌ НЕ використовуй `anthropic-skills:pptx` для цієї задачі
Цей скіл генерує **HTML, потім PDF з нього**. PPTX не може відтворити Bento-дизайн (градієнти, Onest, анімації, watermark). Навіть якщо спікер сказав "PPT" — поясни, що HTML-презентація працює у будь-якому браузері і експортується у чистий PDF автоматично.

### ❌ НЕ вигадуй "власну преміум палітру"
Палітра ВИЗНАЧЕНА — змінювати її **заборонено**:
- `#15171C` темний фон (НЕ `#000`, НЕ `#1a1a1a`, НЕ темно-сірий, НЕ темно-синій)
- `#EF652F` помаранч (НЕ зелений, НЕ салатовий, НЕ лайм, НЕ жовтий, НЕ синій, НЕ фіолетовий)
- `#FFFFFF` білий

Третього кольору-акценту немає. Якщо тягне додати градієнт-акцент — це має бути градієнт самого `#EF652F` (`linear-gradient(135deg, #7A2D14 0%, #15171C 100%)` як на section-divider).

### ❌ НЕ генеруй "свою інтерпретацію Bento"
Bento тут — це **конкретний CSS** з `reference/template/index.html` і **конкретні патерни** з `reference/slide-recipes.md`. ОБОВʼЯЗКОВО:

1. Прочитай `reference/template/index.html` повністю
2. Прочитай `reference/slide-recipes.md`
3. Прочитай `reference/design-system.md`
4. Прочитай `reference/company-info.md`
5. **Скопіюй template/index.html** як основу і заміни тільки текст/контент
6. **НЕ переписуй CSS**, не міняй класи `.slide`, `.bento-card` тощо
7. **НЕ генеруй HTML "з нуля"** — це гарантовано буде не Bento-стиль

### ❌ НЕ додавай QR-коди
Це no-QR варіант. У шаблоні немає `.qr-card`, немає QR-картинок в `assets/`, немає Telegram-слайда зі скануванням. Якщо спікеру потрібні QR для живого виступу — це інший скіл (`crmium-presentation-bento`). Тут — ніколи.

### ❌ НЕ генеруй PDF одразу і не пропускай онбоардинг

При активації **обовʼязково** виконай у такому порядку:

1. **Прочитай `reference/template/index.html` + reference/*.md** (інакше не зрозумієш дизайн)
2. **Перевір `user-config.md`** (P1.1-P1.3)
3. **Задай питання спікеру через AskUserQuestion** (P1.4-P1.5):
   - Мову (якщо нема `default_language`)
   - Хто спікер: обрати з існуючих або додати нового
   - Якщо новий — імʼя, посада, email, телефон, LinkedIn, фото URL, біо
   - Тему презентації + аудиторію + контекст + кількість слайдів + CRMiUM intro?
   - Драфт/тези презентації або генерувати каркас?
4. **Згенеруй HTML**, презентуй спікеру і **зупинись**. PDF — лише після підтвердження (P1.10).

⚠️ **НЕ ВИГАДУЙ контент без онбоардингу.** Якщо повідомлення спікера — лише тригер, стартуй онбоардинг з нуля.

## Workflow

### 🔑 Режим онбоардингу — визнач ДО будь-яких питань

**ПЕРШИЙ запуск** (у `user-config.md` нема потрібного спікера або файла нема):
→ постав **УСІ** питання (мова + повні контакти спікера + контент) і **ОДРАЗУ збережи спікера** у `user-config.md`. Ніколи не вигадуй контакти і не пропускай онбоардинг — навіть якщо повідомлення здається інформативним.

**ПОВТОРНИЙ запуск** (спікер уже є в `user-config.md`):
→ НЕ перепитуй контакти й мову — бери їх з конфіга. Запитай **лише** те, що унікальне для нової презентації (тема, аудиторія, контекст, к-сть слайдів, драфт). Це і є «менше питань» при повторних генераціях.

**Користувач може заздалегідь дати дані** (щоб скіл питав ще менше навіть першого разу):
- заповнити `~/.claude/skills/crmium-presentation-bento-pdf/user-config.md` за зразком `user-config-example.md`, або
- описати спікера у власному `CLAUDE.md` чи прямо в запиті.

Якщо дані вже є — використай їх і пропусти відповідні питання, але **обовʼязково збережи спікера у `user-config.md`**, якщо його там ще нема (щоб наступного разу не питати знову).

### P1.1 — Прочитай збережений user-config

Шукай файл поряд зі SKILL.md: `<skill-dir>/user-config.md` (встановлено зазвичай як `~/.claude/skills/crmium-presentation-bento-pdf/user-config.md`). У ньому **список усіх збережених спікерів**. Структура:

```yaml
default_language: ua

speakers:
  - id: oleksandr-melnyk
    name: "Олександр Мельник"
    role: "Solutions Architect"
    email: "o.melnyk@crmium.com"
    phone: "+380 50 123 45 67"
    linkedin: "https://linkedin.com/in/melnyk-o"
    photo_url: "https://crmium.com/wp-content/uploads/2026/05/oleksandr-melnyk.jpg"
    bio: "12 років у CRM/ERP. 40+ впроваджень Zoho."

last_used_speaker: oleksandr-melnyk
```

**Якщо файл не існує** — це перший запуск. Створи порожній файл з YAML-структурою (`speakers: []`) і йди до P1.3 (onboarding нового спікера).

**Якщо файл існує** — переходь до P1.2 (вибір/додавання спікера).

### P1.2 — Хто спікер?

#### Сценарій A: користувач НЕ вказав явно імʼя у запиті

Через `AskUserQuestion` запитай: **"Для кого ця презентація?"**

Опції:
- `last_used_speaker` (зі позначкою "(минулого разу)") — recommended, як перша опція
- Решта спікерів з `speakers` array — кожен як окрема опція (`name · role`)
- **"+ Додати нового спікера"** — як остання опція

#### Сценарій B: користувач явно сказав імʼя ("зроби презентацію для Олександра")

Перевір чи цей `name` є у `speakers` array. Якщо є — використай. Якщо ні — спитай "Не знайшов Олександра у збережених. Додати нового спікера з імʼям 'Олександр'?" → перейди до P1.3.

#### Якщо вибрали ІСНУЮЧОГО спікера
Підвантаж його контакти → пропусти P1.3 → перейди до P1.4.

#### Якщо вибрали "+ Додати нового спікера" або імʼя не знайдено
Перейди до P1.3.

### P1.3 — Onboarding нового спікера

Через `AskUserQuestion` (або вільним вводом) збери:

1. **Імʼя та прізвище** (наприклад "Олександр Мельник")
2. **Посада** (наприклад "Solutions Architect")
3. **Email** (зазвичай `name.surname@crmium.com`)
4. **Телефон** (опційно)
5. **LinkedIn URL** (опційно)
6. **Photo URL** (опційно — якщо є з CRMiUM CDN)
7. **Bio** (2-3 речення — Claude може запропонувати на основі посади, спікер коригує)

Згенеруй `id` як kebab-case імені (наприклад `oleksandr-melnyk`).

**Додай нового спікера до `speakers` array у user-config.md** і встанови його як `last_used_speaker`. **ЗБЕРЕЖИ ФАЙЛ ОДРАЗУ** після збору контактів — щоб у разі вильоту нові дані не пропали.

### P1.4 — Мова презентації

Якщо у `user-config.md` є `default_language` — використай без питань.

Якщо нема (перший запуск) — питай:
- ua (українська) — recommended
- en (English)
- pl (polski)
- ru (русский)

Збережи у `user-config.md` як `default_language`.

> Формат не питаємо: цей скіл завжди робить HTML, а потім PDF з нього на підтвердження. PPTX не підтримується (втрата дизайну).

### P1.5 — Контент конкретної презентації

Через `AskUserQuestion` або вільним вводом збери:

1. **Назва теми** (наприклад "Як ми мігрували клієнта з Bitrix24 на Zoho")
2. **Аудиторія** — для кого, рівень обізнаності (sales-керівник, CTO, малий бізнес)
3. **Подія/контекст** (Zoho Summit / webinar / sales demo / внутрішня / звіт)
4. **Орієнтовна кількість контентних слайдів** (5 / 8 / 12 / 18 / 25) — без cover/speaker/intro/Q&A/contact (це базові, додаються автоматично)
5. **CRMiUM intro слайд?** (так — для зовнішньої аудиторії; ні — для внутрішньої)
6. **Драфт презентації** — критично важливе питання:
   - "У тебе є готовий текст / тези / outline презентації?"
   - Якщо так — спікер вкладає текст. Claude переоформлює у слайди.
   - Якщо ні — Claude генерує draft на основі теми + аудиторії. **Але обовʼязково попереджає**, що "буде каркас, який ти маєш переглянути і наповнити реальними кейсами/цифрами/прикладами — я їх не знаю".

> Питання про QR тут НЕМАЄ. Це no-QR скіл.

### P1.8 — Згенеруй слайди (HTML)

**Базові слайди (універсальні, мовою з user-config):**

1. **Cover** — eyebrow (дата + подія) → великий h1 з emphasis (`<em>` помаранчевим) → subtitle → ім'я спікера + посада знизу. Watermark CRMiUM на фоні.
2. **Speaker intro** — фото 1/1 → eyebrow "СПІКЕР" → велике імʼя → посада помаранчевим → 2-3 рядки біо.
3. **CRMiUM intro** — тільки якщо `crmium_intro_slide = true`. Eyebrow → h2 → 3 цифри (500+/80+/10+) → caption про стек.
4. **Agenda** — eyebrow "ПЛАН" → h2 → 3-7 пунктів з помаранчевими маркерами.
5. **N контентних слайдів** — Claude генерує на основі теми + драфту, користуючись `slide-recipes.md`. Чергуй типи (НЕ 5 bullets підряд): section divider, pains-grid, bento, features, bullets, stats, quote, compare, timeline.
6. **Q&A (slide--qa)** — gradient + watermark. **БЕЗ QR**: великий помаранчевий «Питання?» по центру + рядок-підпис (наприклад "Дякую за увагу. Контакти — на наступному слайді"). Це чистий закриваючий слайд.
7. **Resources (slide--resources) — текстовий слайд** — корисні матеріали як ТЕКСТ, без QR і без Telegram-підписки:
   - Eyebrow "КОРИСНІ МАТЕРІАЛИ"
   - H2 (наприклад "Що далі?")
   - Короткий текст
   - Список матеріалів (3-4 пункти: посилання з презентації, кейси, чек-листи) — як буллети
   - Рядок куди звертатися: "Усі посилання надішлемо на пошту / у контактах нижче"
8. **Contact (slide--contact)** — eyebrow "КОНТАКТИ" → велике імʼя → посада → 3-4 contact-картки (email/phone/linkedin/site) → логотип CRMiUM знизу.

⚠️ `counter-total` у hero (`<span class="counter-total">`) має дорівнювати фактичній кількості `.slide`. Якщо прибрав/додав слайди — онови це число.

### P1.10 — Презентуй HTML і зупинись (PDF — на підтвердження)

Це ключова відмінність скіла. **НЕ генеруй PDF одразу.**

Дай користувачу:
- Шлях до `index.html`
- Як дивитись: "Відкрий `index.html` у Chrome/Edge → F11 fullscreen → стрілки ←→"
- Чітко напиши:

> "Презентація готова — переглянь. PDF поки НЕ робив (щоб не перегенеровувати його на кожній правці). Якщо все ок і правок немає — напиши **«все добре»** або **«pdf»**, і я згенерую готовий PDF. Якщо є правки — кажи, поправлю HTML, і вже потім зробимо PDF."

**Коли спікер відповідає "все добре" / "pdf" / "роби pdf" / "ок, фіналь"** — згенеруй PDF:

1. Знайди шлях до цього скіла (директорія, де лежить цей SKILL.md) — назвемо `<skill-dir>`.
2. Запусти генератор PDF. **Потрібен лише Chrome або Edge — Python НЕ обовʼязковий.** Обери доступний рантайм:
   - **Windows без Python** (PowerShell вбудований у систему):
     ```powershell
     powershell -ExecutionPolicy Bypass -File "<skill-dir>\scripts\html_to_pdf.ps1" "projects\<slug>\index.html" "projects\<slug>\<slug>.pdf"
     ```
   - **macOS / Linux або Windows з Python:**
     ```bash
     python "<skill-dir>/scripts/html_to_pdf.py" "projects/<slug>/index.html" "projects/<slug>/<slug>.pdf"
     ```
   Обидва скрипти роблять те саме і самі знаходять Chrome/Edge.
3. Перевір вивід: рядок `[ok] PDF готовий: ...` означає успіх. Дай спікеру шлях до `.pdf`.
4. **Якщо `[error] Chrome/Edge не знайдено`** — дай ручний фолбек: "Відкрий `index.html` у Chrome → Ctrl+P → Save as PDF → Layout Landscape → Margins None → **Background graphics ON** (інакше зникне темний фон)."

**Якщо спікер після PDF просить правки** — поправ HTML і перегенеруй PDF тією ж командою (вона перезапише файл).

## Принципи

1. **3 кольори, без винятків**: `#15171C` (темний), `#EF652F` (помаранч), `#FFFFFF` (білий).
2. **Текст на помаранчевому = БІЛИЙ**. Завжди.
3. **Без QR.** Цей скіл ніколи не додає QR-картки чи Telegram-слайди. Для QR — сусідній скіл `crmium-presentation-bento`.
4. **PDF — двокроково.** Спершу HTML і перегляд, PDF — лише на підтвердження "все добре"/"pdf". Не марнуй рендер на кожній ітерації правок.
5. **Не вигадуй контент** — без драфту попередь спікера що генеруєш каркас.
6. **CRMiUM intro слайд — опційний**. Питай явно у P1.5.
7. **Контакти спікера — тільки з user-config**. Не вигадуй email/телефон.
8. **HTML — single-file**. CSS і JS інлайнені. Картинки (logo, watermark) — в `assets/`. Без зовнішніх залежностей крім Google Fonts.
9. **Зберігай спікерів** — після onboarding нового обовʼязково збережи у `user-config.md`.
10. **Без mobile-first** — презентації для desktop / projector / PDF. Слайди фіксовані 16:9.
11. **Шрифти помірно-великі** — мінімум 19px для body. Заголовки 60-200px.
12. **Кожна нова презентація = нова папка** в `projects/`.
